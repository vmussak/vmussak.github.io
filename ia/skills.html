Skills são capacidades sob demanda

Uma Skill empacota instruções, scripts, referências e ferramentas para uma tarefa específica. Em vez de jogar todo o conhecimento no prompt principal, o agente carrega só o que precisa, quando precisa.

①
Metadados
Nome e descrição ficam disponíveis para o agente decidir se a Skill é relevante.

          
②
Instruções
O agente carrega o SKILL.md apenas quando o gatilho combina com a tarefa.

          
③
Recursos
Scripts, templates, exemplos, APIs, SQL e CLIs entram só quando necessários.

        

      

    


    

      
🧩
Carregamento em 3 níveis
Primeiro o agente vê metadados, depois instruções específicas e, por fim, recursos pesados como scripts, referências e templates.

      
🎯
Acionamento sob demanda
A Skill só entra quando a descrição combina com a tarefa. Isso evita carregar conhecimento irrelevante em toda conversa.

      
🪟
Menos context window
O prompt inicial fica menor, mais limpo e com menos ruído. A janela é preenchida progressivamente, conforme a necessidade.

      
⚙️
Scripts em qualquer linguagem
A Skill pode orientar a execução de Python, Bash, PowerShell, Node, C#, SQL ou qualquer linguagem disponível no ambiente.

      
🔌
APIs, SQL e CLI
Ela pode padronizar chamadas para APIs internas, queries SQL, comandos de terminal, ferramentas de deploy, lint, testes e automações.

      
📦
Conhecimento reutilizável
Boas práticas, checklists, comandos e padrões deixam de ficar espalhados em prompts enormes e viram capacidades versionáveis.
