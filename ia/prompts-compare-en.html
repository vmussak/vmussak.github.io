Core idea

        

          Tool calling is not "free". Each call typically adds to the context:
          the user message, the LLM reasoning / planning, the tool parameters,
          the tool response, and a new interpretation before the next action.
        

        
    
          
  • Vague prompt: forces discovery of directories, files and tests.
    • 
          
  • Refined prompt: narrows the search space and goes almost straight to the patch.
    • 
          
  • Less irrelevant context: lower cost, faster responses and fewer mistakes.
    • 
        

        

          Better prompt
          Include the file, the symptom, the business rule, the expected test and any change constraints.
